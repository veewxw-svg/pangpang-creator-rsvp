# PangPang 博主探店预约系统上线部署说明

这个文件夹现在已经是一个可部署的网站服务。

## 推荐部署方式

推荐用 Render，选择 Docker 部署。

上线后会得到一个真正的网址，例如：

```text
https://pangpang-creator-rsvp.onrender.com
```

这个网址手机和电脑都能打开，不需要你的电脑一直开着。

## Render 需要的设置

项目根目录：

```text
07_执行交付_当前可发/PangPang_博主探店预约小工具
```

部署方式：

```text
Docker
```

持久化数据盘：

```text
mount path: /var/data
size: 1GB
```

系统会把正式记录存在：

```text
/var/data/records.json
```

## 邮件通知

系统已经接好 Resend 邮件接口，但默认关闭。

等你有 Resend API Key 后，在 Render 环境变量里设置：

```text
SEND_EMAILS=1
RESEND_API_KEY=你的 Resend API Key
NOTIFY_FROM=PangPang <通知发件邮箱>
NOTIFY_TO=老板邮箱,店长邮箱,营销邮箱
PUBLIC_URL=你的正式网站网址
```

每次保存记录后，系统会生成全局 PNG 并发邮件。

## 本地开发

```bash
node server.mjs
```

打开：

```text
http://127.0.0.1:8787/
```

全局 PNG：

```text
http://127.0.0.1:8787/api/report.png
```
